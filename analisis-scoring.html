⚙️ Logika Scoring di Backend

Bagaimana Sistem Menghitung Nilai?

Berikut adalah logika yang digunakan di Code.gs untuk menghitung nilai setiap tipe soal:

🧮 PGK Mode Simple - Algoritma Scoring

// Pseudocode dari Code.gs line 212-238 if (q.tipe === 'pgk' && cfg.pgk.mode === 'simple') { totalTrue = hitung jumlah opsi yang seharusnya benar correctSelected = hitung opsi benar yang dipilih siswa wrongSelected = hitung opsi salah yang dipilih siswa if (tidak ada pilihan) { score = simpleBlankPoints } else if (correctSelected == totalTrue && wrongSelected == 0) { // SEMUA BENAR: setiap opsi benar dipilih, tidak ada opsi salah yang dipilih score = simpleAllCorrectPoints // Anda: 1 } else if (correctSelected == 0) { // SALAH SEMUA: tidak ada opsi benar yang dipilih score = simpleAllWrongPoints // Anda: 0 } else { // BENAR SEBAGIAN: ada opsi benar, tapi tidak semua / ada opsi salah terseleksi score = simplePartialPoints // Anda: 0 } return score (clamped between minPoints & maxPoints) }

🧮 BS Mode Simple - Algoritma Scoring

// Pseudocode dari Code.gs line 243-271 if (q.tipe === 'bs' && cfg.bs.mode === 'simple') { statements = array pernyataan di soal correctCount = hitung berapa pernyataan dijawab benar totalStatements = jumlah pernyataan if (tidak ada jawaban) { score = simpleBlankPoints } else if (correctCount == totalStatements) { // SEMUA BENAR: semua pernyataan dijawab benar score = simpleAllCorrectPoints // Anda: 1 } else if (correctCount == 0) { // SALAH SEMUA: tidak ada pernyataan yang dijawab benar score = simpleAllWrongPoints // Anda: 0 } else { // BENAR SEBAGIAN: ada yang benar, ada yang salah score = simplePartialPoints // Anda: 0 } return score (clamped between minPoints & maxPoints) }

⚠️ Penting: Score Clamping

Nilai akhir selalu di-clamp (dikurung) antara minPoints dan maxPoints:

finalScore = Math.max(minPoints, Math.min(maxPoints, score))

Contoh: Jika simpleAllWrongPoints=0 dan minPoints=0, nilai akhir tetap 0. ✓
Jika simpleAllCorrectPoints=1 dan maxPoints=10, nilai akhir tetap 1. ✓
